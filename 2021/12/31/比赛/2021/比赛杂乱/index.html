phar文件结构有四部分组成

  • stub ：phar文件标识
  • a manifest describing the contents：被压缩文件的权限、属性等信息
  • the file contents：被压缩文件的内容
  • [optional] a signature for verifying Phar integrity (phar file format only)：签名

stub格式为xxx<?php xxx; __HALT_COMPILER();?>，前面内容不限，但必须以__HALT_COMPILER();?>来结尾，否则phar扩展将无法识别这个文件为phar文件

a manifest describing the contents：phar文件本质上是一种压缩文件，其中每个被压缩文件的权限、属性等信息都放在这部分。这部分还会以序列化的形式存储用户自定义的meta-data，这是上述攻击手法最核心的地方

但是这题提供一个新思路

文件里不在有<?php __HALT_COMPILER();?> 而且有垃圾字符也不影响
